## 1. Current Direction

### Direction Name

**Personal Product System Portfolio**

### One-Line Definition

A product-led UX/UI portfolio that feels like a personal design system: structured, useful, slightly playful, technically aware, and focused on making repeated workflows easier for users and clearer for teams.

### Core Positioning

> **Easy to work with. Serious about the details.**

### Supporting Line

> I design practical digital products that make repeated workflows easier, clearer, and less annoying — for users and teams.

### Core Visual Narrative

> **Complexity → Structure → Clarity**

### Final Experience Statement

> **Structured enough to trust. Human enough to remember.**

## 7. Global Non-Negotiable Rules

### Content Rules

- Do not invent metrics.
- Do not claim impact without evidence.
- Do not overstate the designer's role.
- Do not rewrite the core positioning without explicit direction.
- Do not change the project hierarchy.
- Do not use generic UX buzzwords as proof.

Avoid words such as:

```text
seamless
intuitive
delightful
pixel-perfect
visionary
innovative
beautiful experiences
```

### Confidentiality Rules

Do not expose:

- real client names
- real client logos
- real customer data
- real customer names
- real emails
- phone numbers
- customer IDs
- real campaign names
- internal URLs
- API keys
- access tokens
- backend architecture
- confidential business rules
- pricing / revenue / contract details

Use instead:

- dummy data
- sanitized reconstruction
- recreated screens
- illustrative mockups
- abstract workflow diagrams
- anonymized screenshots

### Approved Labels

Use these labels inside case studies when needed:

```text
Sanitized reconstruction
Recreated for portfolio clarity
Illustrative mockup
Workflow mockup
Visual exploration
Anonymized screen
```

Do not over-label homepage project cards.

## 8. Visual Direction

The visual style should feel like:

> **warm technical editorial product-design documentation**

Use:

- warm off-white background
- warm-white / white cards
- charcoal text
- thin neutral borders
- precise grid
- calm spacing
- restrained accent color
- state chips
- decision notes
- constraint tags
- annotation pins
- connector lines
- reusable UI fragments

Avoid:

- neon
- sci-fi HUD
- glossy 3D
- glassmorphism
- dark cinematic website
- generic SaaS dashboard clone
- stock business people
- cute cartoon illustration
- random abstract blobs
- fake metrics
- developer terminal aesthetic

---

## 9. Interaction Principle

> Motion should explain, not decorate.

Good motion:

- scattered workflow fragments align into structure
- connector lines reveal relationships
- hover reveals before → decision → after
- scroll reveals complexity becoming clearer
- reduced-motion fallback keeps content readable

Avoid:

- decorative floating objects
- over-engineered scroll tricks
- heavy animation that hides content
- hover-only critical information

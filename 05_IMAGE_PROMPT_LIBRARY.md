# 05 — Image Prompt Library
## Portfolio Visual Asset Prompts

> **Purpose**  
> Use this file for Midjourney, Firefly, Ideogram, DALL·E, or other image generators.  
> These prompts create visual direction assets, not final website UI.

---

## 1. General Image Generation Rules

Use short readable labels only.

Image generators often distort long text, so:

- keep all labels short
- avoid long paragraphs inside the image
- use visual structure first, text second
- use placeholder labels when needed
- leave space for manual text replacement in Figma later

All images must be:

- confidentiality-safe
- dummy-data only
- no real logos
- no real customer names
- no real campaign names
- no fake metrics

---

## 2. Master Visual Strategy Board

```text
Create a text-light visual strategy board for a UX/UI designer portfolio called Personal Product System Portfolio.

Title concept: Warm Workflow System Board.

The board should summarize the art direction before final UI design. It should feel like warm technical editorial product-design documentation.

Use a warm off-white background, white or warm-white cards, thin neutral gray borders, charcoal text, restrained blue accent, precise grid, calm spacing, small annotation labels, connector lines, state chips, decision cards, constraint tags, workflow arrows, and reusable UI fragments.

Arrange the board into six spacious zones:

1. Homepage hierarchy map
2. Hero workflow visual mood sample
3. CDP reusable search/filter thumbnail
4. Service Transaction POS dual-screen thumbnail
5. Case study visual system
6. Motif library with confidentiality rules

The board should communicate:

Complexity → Structure → Clarity

Do not create a final website mockup. This should look like an art direction board, not a finished landing page.

Use short readable labels only. Leave space for manual text replacement.

No real logos, no real data, no fake metrics, no generic SaaS dashboard, no neon, no sci-fi HUD, no glossy 3D, no glassmorphism, no stock business people.
```

Recommended aspect ratio:

```text
4:3 for art direction review
3:2 for visual system board
16:9 for presentation slide usage
```

## 3. Homepage Hero Visual

```text
Create a homepage hero visual for a UX/UI designer portfolio.

The hero should support the headline:

“Easy to work with. Serious about the details.”

Composition: spacious left headline area and right-side workflow system board.

The visual board should show scattered workflow fragments becoming a readable product system. Use task cards, state chips, decision notes, constraint tags, tiny UI fragments, thin connector lines, subtle grid alignment, and small annotation labels.

The mood should be warm technical editorial product-design documentation: off-white background, white cards, charcoal text, thin gray borders, restrained accent color, calm spacing.

The hero should feel practical, structured, warm, product-led, and detail-oriented. It should communicate that the designer makes repeated workflows clearer for users and teams.

Do not make it look like a generic dashboard screenshot. Do not use neon, sci-fi HUD, glossy 3D, glassmorphism, dark cinematic style, abstract blobs, stock people, or fake metrics.
```

## 9. Quality Check

Before accepting generated visuals, ask:

1. Does it communicate Complexity → Structure → Clarity?
2. Does it support “Easy to work with. Serious about the details.”?
3. Is the homepage hierarchy clear?
4. Does CDP look like reusable B2B workflow logic?
5. Does POS look like service transaction workflow?
6. Are all visuals confidentiality-safe?
7. Are there no unsupported metrics?
8. Does every visual element have a reason?
9. Does the image feel warm, technical, editorial, and product-led?
10. Can this image guide the next design step?

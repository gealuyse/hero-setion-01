# 03 — Art Direction Brief
## Warm Workflow System

> **Purpose**  
> Use this file for visual design, AI image generation, wireframe direction, visual system decisions, and art direction review.

---

## 1. Core Visual Direction

The portfolio should feel like:

> **warm technical editorial product-design documentation**

It should look like:

> a system with life in it — structured enough to trust, human enough to remember.

---

## 2. Core Narrative

> **Complexity → Structure → Clarity**

This story should appear across:

- homepage
- project thumbnails
- case study visuals
- motion
- interaction patterns
- motif library

---

## 3. Meaning of Each Stage

### Complexity

Complexity should be shown as product-related confusion, not abstract chaos.

Use:

- scattered workflow cards
- disconnected modules
- unclear state labels
- hidden rules
- repeated tasks
- table fragments
- form fragments
- modal fragments
- filter fragments
- action fragments
- notes like "people guess here" or "logic hidden"

Avoid:

- random abstract blobs
- sci-fi panels
- chaotic decoration
- glossy abstract 3D

### Structure

Structure is where design makes product logic visible.

Use:

- aligned grids
- grouped modules
- connector lines
- state chips
- decision cards
- constraint tags
- reusable UI fragments
- visible workflow paths

### Clarity

Clarity is the result.

Show:

- what task is being completed
- which action matters most
- what state the system is in
- what decision was made
- what pattern is reusable
- how teams can discuss the same flow without guessing

Clarity should not be represented only by a polished final screen.

It should be represented by:

> a system that explains itself.

## 6. Hero Direction

### Recommended Hero Concept

**Workflow Fragments Becoming a System**

### What It Communicates

The designer turns scattered tasks, states, requirements, and decisions into a clearer product system.

### Composition

- left side: headline, supporting line, role line, CTA
- right side: workflow system board
- cards, task notes, state chips, decision cards, constraint tags, tiny UI fragments
- connector lines showing relationships
- structured but not rigid

### Visual Elements

- workflow cards
- task labels
- state chips
- decision notes
- constraint tags
- thin connector lines
- dashed alignment guides
- table row fragments
- filter chips
- form fields
- button groups

### Motion Potential

- fragments gently align into a grid
- connector lines draw in
- important labels fade in
- hover highlights one workflow path
- reduced-motion fallback shows final aligned state only

### Risks

Avoid making it:

- sci-fi
- neon
- too dense
- too decorative
- a fake dashboard
- louder than the headline

## 7. Selected Work Visual Direction

Selected Work should not behave like a gallery of pretty thumbnails.

It should answer:

> What repeated task was involved, and what became clearer?

For V1, show two main project cards:

1. Customer Data Platform / CDP
2. Service Transaction POS

### CDP Thumbnail

Visual story:

> A complex B2B marketing system becomes clearer through reusable search/filter logic and visible workflow structure.

Composition:

- reusable search/filter module at the center
- Basic search → Advanced search → Condition builder
- connected modules around it:
  - customer data table
  - advanced filter panel
  - condition builder
  - audience segment chips
  - consent requirement card
  - campaign setup block
  - journey flow nodes

Avoid:

- generic analytics dashboard
- KPI-heavy dashboard
- fake charts / fake metrics

### POS Thumbnail

Visual story:

> Modernizing a familiar service transaction workflow without breaking staff habits.

Composition:

- two interface frames:
  - 800×600 compact screen
  - 1920×1080 wide screen
- same service transaction flow adapted across sizes
- service search
- service selection
- customer input
- summary panel
- primary action
- confirmation / receipt
- warning / error / success states

Avoid:

- restaurant POS
- beverage ordering
- retail checkout
- product-sales POS

## 10. Visual System Recommendations

### Color Direction

Base:

- warm off-white page background
- charcoal / near-black text
- warm-white cards
- soft gray borders

Accent options:

- muted rust as expressive primary direction
- ink-blue as conservative fallback
- olive as third option

### Typography

Recommended:

- General Sans + IBM Plex Mono

Safe fallback:

- Inter + IBM Plex Mono

Typography should feel:

- clear
- modern
- editorial
- product-minded
- readable on mobile

### Layout

Recommended:

- mobile-first
- 12-column desktop grid
- 8px spacing system
- calm whitespace
- modular cards
- strong headline hierarchy
- project cards as product modules

### Card Style

- 12px radius
- hairline border
- no heavy shadows
- warm-white surface
- precise internal spacing

---

## 11. Final Art Direction Check

Before accepting any visual, ask:

1. Does it communicate Complexity → Structure → Clarity?
2. Does it support “Easy to work with. Serious about the details.”?
3. Is the homepage hierarchy clear?
4. Does CDP look like reusable B2B workflow logic, not generic analytics?
5. Does POS look like service transaction workflow, not retail or restaurant POS?
6. Are the visuals confidentiality-safe?
7. Are there no unsupported metrics?
8. Does every visual element have a reason?
9. Does the image feel warm, technical, editorial, and product-led?
10. Can this image guide the next design step?
